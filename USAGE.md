# Frontend Tech Blog 使用流程指南

## 快速开始

### 第一次使用

#### 1. 环境准备
确保你的电脑已安装以下软件：
- **Node.js** 18.0+ （建议使用 18.17.0 或更高版本）
- **npm** 9.0+ 或 **pnpm** 8.0+
- **Git**（可选，用于版本控制）

检查环境：
```bash
node --version
npm --version
```

#### 2. 克隆或下载项目
```bash
cd D:\v3\frontend-blog
```

#### 3. 安装依赖
```bash
npm install
```
等待依赖安装完成（第一次可能需要 3-5 分钟）

#### 4. 启动开发服务器
```bash
npm run dev
```

#### 5. 访问博客
打开浏览器访问：`http://localhost:5173`

## 项目部署

### 本地预览
```bash
npm run build
npm run preview
```

### 部署到服务器

#### 1. 构建生产版本
```bash
npm run build
```

#### 2. 部署到服务器
有多种部署方式：

**方式一：传统服务器**
- 使用 FTP/SFTP 上传 `dist/` 目录
- 配置 Nginx 或 Apache

**方式二：云服务**
- 部署到 Vercel：`vercel deploy`
- 部署到 Netlify：`netlify deploy`
- 部署到 GitHub Pages

**方式三：Docker 部署**
```bash
docker build -t frontend-blog .
docker run -p 80:80 frontend-blog
```

### 更新内容流程

1. 在本地修改文章
2. 运行 `npm run build` 构建
3. 部署更新到服务器

## 常见问题

### Q1: 如何更新文章内容？
A: 直接编辑 `posts/` 目录下的 `.md` 文件

### Q2: 如何添加新文章？
A: 在 `posts/` 目录下创建新的 `.md` 文件

### Q3: 如何导出为 PDF？
A: 使用浏览器打印功能，选择"保存为 PDF"

### Q4: 如何分享给同事？
A: 可以部署到服务器，或直接分享文件

### Q5: 端口被占用怎么办？
A: 修改 `vite.config.ts` 中的端口配置
