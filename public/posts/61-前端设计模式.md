## 一、创建型模式

### 1. 单例模式 (Singleton)

**意图**：保证一个类只有一个实例，并提供一个全局访问点。

**应用场景**：
- Vuex/Pinia 状态管理
- 全局 Loading 组件
- 日志系统
- 缓存管理

**JavaScript 实现**：

```javascript
class Singleton {
  constructor() {
    if (Singleton.instance) {
      return Singleton.instance
    }
    Singleton.instance = this
    this.data = []
  }
  
  addData(item) {
    this.data.push(item)
  }
}

const instance1 = new Singleton()
const instance2 = new Singleton()
console.log(instance1 === instance2) // true
```

### 2. 工厂模式 (Factory)

**意图**：定义一个创建对象的接口，让子类决定实例化哪个类。

**应用场景**：
- 组件工厂
- API 适配器工厂
- 表单控件工厂

**JavaScript 实现**：

```javascript
function createButton(type) {
  const buttonMap = {
    primary: () => '<button class="primary">确定</button>',
    danger: () => '<button class="danger">删除</button>'
  }
  return buttonMap[type]?.() || '<button>默认</button>'
}
```

### 3. 建造者模式 (Builder)

**意图**：将一个复杂对象的构建与表示分离。

**应用场景**：
- 复杂表单构建
- SQL 查询构建器
- 配置对象构建

**TypeScript 实现**：

```typescript
class RequestBuilder {
  private url = ''
  private method: 'GET' | 'POST' = 'GET'
  private headers: Record<string, string> = {}
  
  setUrl(url: string) { this.url = url; return this }
  setMethod(method: 'GET' | 'POST') { this.method = method; return this }
  build() { return { url: this.url, method: this.method, headers: this.headers } }
}

const request = new RequestBuilder()
  .setUrl('/api/users')
  .setMethod('POST')
  .build()
```

## 二、结构型模式

### 1. 装饰器模式 (Decorator)

**意图**：动态地给对象添加一些额外的职责。

**TypeScript 实现**：

```typescript
function log(target: any, key: string, descriptor: PropertyDescriptor) {
  const original = descriptor.value
  descriptor.value = function(...args: any[]) {
    console.log(`调用 ${key}:`, args)
    return original.apply(this, args)
  }
  return descriptor
}

class Calculator {
  @log
  add(a: number, b: number): number {
    return a + b
  }
}
```

---

### 2. 适配器模式 (Adapter)

**意图**：将一个类的接口转换成客户希望的另一个接口。

```typescript
class OldAPI {
  getUser() { return { user_name: '张三', user_age: 25 } }
}

class Adapter {
  constructor(private api: OldAPI) {}
  getUser() {
    const old = this.api.getUser()
    return { name: old.user_name, age: old.user_age }
  }
}
```

### 3. 代理模式 (Proxy)

**意图**：为其他对象提供一种代理以控制对这个对象的访问。

**缓存代理示例**：

```typescript
function cached<T extends (...args: any[]) => any>(fn: T) {
  const cache = new Map<string, any>()
  return function(...args: any[]) {
    const key = JSON.stringify(args)
    if (cache.has(key)) return cache.get(key)
    const result = fn.apply(this, args)
    cache.set(key, result)
    return result
  } as T
}
```

## 三、行为型模式

### 1. 观察者模式 (Observer)

**意图**：定义对象间的一种一对多依赖关系。

```typescript
class EventBus {
  private handlers: Map<string, Function[]> = new Map()
  
  on(event: string, handler: Function) {
    if (!this.handlers.has(event)) this.handlers.set(event, [])
    this.handlers.get(event)!.push(handler)
  }
  
  emit(event: string, ...args: any[]) {
    this.handlers.get(event)?.forEach(h => h(...args))
  }
}
```

---

### 2. 策略模式 (Strategy)

**意图**：定义一系列算法，使它们可以相互替换。

```typescript
interface ValidationStrategy {
  validate(value: any): boolean
  message: string
}

class RequiredValidator implements ValidationStrategy {
  message = '必填'
  validate(v: any) { return v !== null && v !== undefined }
}

class EmailValidator implements ValidationStrategy {
  message = '邮箱格式错误'
  validate(v: any) { return /^[^\s@]+@[^\s@]+\.[^\s@]+$/.test(v) }
}
```

---

### 3. 命令模式 (Command)

**意图**：将请求封装成对象，支持撤销/重做。

```typescript
interface Command { execute(): void; undo(): void }

class SetTextCommand implements Command {
  constructor(private editor: any, private newText: string, private oldText: string) {}
  execute() { this.editor.text = this.newText }
  undo() { this.editor.text = this.oldText }
}
```

---

### 4. 迭代器模式 (Iterator)

**意图**：顺序访问集合元素，不暴露内部结构。

```typescript
class LinkedList<T> {
  *[Symbol.iterator](): Generator<T> {
    let current = this.head
    while (current) {
      yield current.value
      current = current.next
    }
  }
}
```

## 四、架构型模式

### 1. MVC 模式

```
┌─────────┐      ┌────────────┐      ┌─────────┐
│  View   │◄────►│ Controller │◄────►│  Model  │
└─────────┘      └────────────┘      └─────────┘
```

### 2. MVVM 模式

```
┌─────────┐      ┌───────────┐      ┌─────────┐
│  View   │◄────►│ ViewModel │◄────►│  Model  │
└─────────┘      └───────────┘      └─────────┘
```
Vue/Angular 采用此模式，实现数据双向绑定。

### 3. Flux 模式

```
Action → Dispatcher → Store → View
```
Redux/Vuex 基于此模式，单向数据流。
