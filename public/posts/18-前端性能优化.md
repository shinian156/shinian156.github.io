# 前端性能优化

## 一、性能优化指标

### 1. 核心指标（Web Vitals）

#### LCP（Largest Contentful Paint）- 最大内容绘制
**定义**：页面主要内容加载完成的时间。

**标准**：
- 良好：< 2.5s
- 一般：2.5s - 4s
- 较差：> 4s

**优化方法**：
```javascript
// 1. 优化图片加载
<img loading="lazy" src="image.jpg" width="800" height="600" />

// 2. 预加载关键资源
<link rel="preload" href="font.woff2" as="font" type="font/woff2" crossorigin>

// 3. 避免阻塞渲染
<script defer src="app.js"></script>
<script async src="analytics.js"></script>
```

**解释**：LCP 衡量用户感知到的加载速度。优化图片、预加载关键资源、避免阻塞渲染可以显著降低 LCP。

---

#### FID（First Input Delay）- 首次输入延迟
**定义**：用户首次与页面交互（点击、输入）到浏览器响应的时间。

**标准**：
- 良好：< 100ms
- 一般：100ms - 300ms
- 较差：> 300ms

**优化方法**：
```javascript
// 1. 减少主线程工作量
// 避免长任务
function heavyTask() {
  // ❌ 错误：阻塞主线程
  for (let i = 0; i < 10000000; i++) {
    // 大量计算
  }
}

// ✅ 正确：拆分任务或使用 Web Worker
function splitTask() {
  const chunkSize = 10000
  let i = 0

  function processChunk() {
    const end = Math.min(i + chunkSize, 10000000)
    for (; i < end; i++) {
      // 处理
    }

    if (i < 10000000) {
      requestAnimationFrame(processChunk)
    }
  }

  processChunk()
}

// 2. 延迟非关键 JavaScript
document.addEventListener('DOMContentLoaded', () => {
  // 非关键代码在这里执行
})
```

**解释**：FID 衡量交互响应速度。减少主线程阻塞任务、延迟执行非关键代码可以降低 FID。

---

#### CLS（Cumulative Layout Shift）- 累积布局偏移
**定义**：页面元素在加载过程中发生的意外布局移动。

**标准**：
- 良好：< 0.1
- 一般：0.1 - 0.25
- 较差：> 0.25

**优化方法**：
```html
<!-- 1. 为图片预留空间 -->
<div style="width: 800px; height: 600px;">
  <img src="image.jpg" width="800" height="600" />
</div>

<!-- 2. 使用 aspect-ratio -->
<img src="image.jpg" style="aspect-ratio: 4/3;" />

<!-- 3. 避免插入内容到已渲染元素上方 -->
<!-- ❌ 错误 -->
<div id="content">
  已渲染内容
</div>
<script>
  document.getElementById('content').insertAdjacentHTML('beforebegin', '<div>新内容</div>')
</script>

<!-- ✅ 正确：使用占位符或预留空间 -->
<div id="placeholder" style="height: 100px;"></div>
<div id="content">已渲染内容</div>
```

**解释**：CLS 衡量视觉稳定性。为图片预留空间、避免动态插入内容可以减少布局偏移。

---

### 2. 其他重要指标

#### TTI（Time to Interactive）- 可交互时间
**定义**：页面完全可交互的时间。

**优化方法**：
```javascript
// 1. 延迟加载非关键 JavaScript
const observer = new IntersectionObserver((entries) => {
  entries.forEach(entry => {
    if (entry.isIntersecting) {
      const script = document.createElement('script')
      script.src = 'lazy-script.js'
      document.body.appendChild(script)
      observer.unobserve(entry.target)
    }
  })
})

observer.observe(document.querySelector('.lazy-load'))

// 2. 使用 Web Worker 处理计算密集型任务
const worker = new Worker('worker.js')
worker.postMessage({ data: 'large' })
worker.onmessage = (e) => {
  console.log(e.data)
}
```

**解释**：TTI 衡量页面何时完全可用。延迟加载非关键代码、使用 Web Worker 处理复杂计算可以缩短 TTI。

---

#### FCP（First Contentful Paint）- 首次内容绘制
**定义**：页面首次渲染任何内容的时间。

**优化方法**：
```html
<!-- 1. 内联关键 CSS -->
<style>
  /* 首屏样式 */
  .hero { display: flex; }
  .title { font-size: 2rem; }
</style>

<!-- 2. 预加载关键资源 -->
<link rel="preload" href="critical.css" as="style">
<link rel="preload" href="hero-image.jpg" as="image">
```

**解释**：FCP 衡量首次视觉反馈。内联关键 CSS、预加载关键资源可以加快 FCP。

## 二、资源优化

### 1. 图片优化

#### 图片格式选择
```javascript
// WebP：推荐使用，体积更小
<img src="image.webp" onerror="this.src='image.jpg'" />

// 响应式图片
<picture>
  <source srcset="image-small.webp" media="(max-width: 600px)">
  <source srcset="image-medium.webp" media="(max-width: 1200px)">
  <img src="image-large.webp" alt="响应式图片">
</picture>

// 懒加载
<img loading="lazy" src="image.jpg" alt="懒加载图片">
```

**解释**：
- WebP 格式比 JPEG 小 25-35%，比 PNG 小 26%
- 响应式图片根据屏幕尺寸加载合适大小的图片
- 懒加载延迟加载非首屏图片，减少初始加载时间

---

#### 图片压缩
```javascript
// 使用 sharp 库压缩图片（Node.js）
const sharp = require('sharp')

// 压缩为 WebP
await sharp('input.jpg')
  .webp({ quality: 80 })
  .toFile('output.webp')

// 压缩为 JPEG
await sharp('input.png')
  .jpeg({ quality: 85, progressive: true })
  .toFile('output.jpg')

// 调整尺寸
await sharp('input.jpg')
  .resize(800, 600, {
    fit: 'inside',
    withoutEnlargement: true
  })
  .toFile('output.jpg')
```

**解释**：
- 压缩图片可以减少 50-80% 的文件大小
- progressive JPEG 逐步显示，提升用户体验
- 适当调整尺寸避免加载过大图片

### 2. CSS 优化

#### 减少和压缩 CSS
```css
/* ❌ 错误：重复代码 */
.button {
  padding: 10px 20px;
  border-radius: 5px;
}

.primary-button {
  padding: 10px 20px;
  border-radius: 5px;
  background: blue;
}

/* ✅ 正确：合并重复代码 */
.button {
  padding: 10px 20px;
  border-radius: 5px;
}

.primary-button {
  @extend .button;
  background: blue;
}
```

**解释**：减少重复代码可以降低 CSS 文件大小。

---

#### 关键 CSS 内联
```html
<!DOCTYPE html>
<html>
<head>
  <!-- ✅ 内联关键 CSS -->
  <style>
    .hero { display: flex; justify-content: center; }
    .title { font-size: 2rem; font-weight: bold; }
  </style>

  <!-- 其他 CSS 异步加载 -->

  <noscript><link rel="stylesheet" href="styles.css"></noscript>
</head>
</html>
```

**解释**：内联关键 CSS 可以避免 CSS 阻塞渲染，加快首屏显示。

---

#### 避免 CSS 阻塞
```html
<!-- ❌ 错误：阻塞渲染 -->
<link rel="stylesheet" href="styles.css">

<!-- ✅ 正确：异步加载 -->
<link rel="preload" href="styles.css" as="style" onload="this.onload=null;this.rel='stylesheet'">
```

**解释**：异步加载 CSS 可以避免阻塞渲染。

### 3. JavaScript 优化

#### 减少 JavaScript 体积
```javascript
// ❌ 错误：引入整个库
import _ from 'lodash'

// ✅ 正确：按需引入
import { debounce } from 'lodash-es'

// ✅ 或使用 tree-shaking
import { debounce } from 'lodash'
```

**解释**：按需引入可以减少打包体积 50-90%。

---

#### 代码分割
```javascript
// 路由懒加载（Vue Router）
const Home = () => import('@/views/Home.vue')
const About = () => import('@/views/About.vue')

const routes = [
  { path: '/', component: Home },
  { path: '/about', component: About }

function loadModule() {
  import('./module.js').then(module => {

  })

const LazyComponent = React.lazy(() => import('./LazyComponent'))
```

**解释**：代码分割将代码拆分成小块，按需加载，减少初始加载体积。

---

#### Tree Shaking
```javascript
// utils.js

// ✅ subtract 会被 tree-shaking 掉
```

**解释**：Tree Shaking 会移除未使用的代码，减少打包体积。

## 三、渲染优化

### 1. 减少重排和重绘

#### 重排（Reflow）
```javascript
// ❌ 错误：多次触发重排
const element = document.getElementById('box')
element.style.width = '100px'
element.style.height = '100px'
element.style.background = 'red'

// ✅ 正确：批量修改
const element = document.getElementById('box')
element.style.cssText = 'width: 100px; height: 100px; background: red;'

// ✅ 或使用 class
element.classList.add('box-styles')
```

**解释**：
- 重排是浏览器重新计算元素的位置和大小
- 批量修改样式可以减少重排次数

---

#### 重绘（Repaint）
```javascript
// ❌ 错误：频繁修改样式
for (let i = 0; i < 1000; i++) {
  element.style.left = i + 'px'
}

// ✅ 正确：使用 requestAnimationFrame
let i = 0
function animate() {
  element.style.left = i + 'px'
  i++
  if (i < 1000) {
    requestAnimationFrame(animate)
  }
}
animate()
```

**解释**：
- 重绘是浏览器重新绘制元素外观
- requestAnimationFrame 会在浏览器下一帧重绘前执行，避免过度渲染

---

### 2. 虚拟滚动
```javascript
// 使用 vue-virtual-scroller
<template>
  <RecycleScroller
    :items="items"
    :item-size="50"
    key-field="id"
    v-slot="{ item }"
  >
    <div>{{ item.name }}</div>
  </RecycleScroller>
</template>

<script>
import { RecycleScroller } from 'vue-virtual-scroller'

export default {
  components: { RecycleScroller },
  data() {
    return {
      items: Array.from({ length: 10000 }, (_, i) => ({
        id: i,
        name: `Item ${i}`
      }))
    }
  }
}
</script>
```

**解释**：
- 虚拟滚动只渲染可见区域的元素
- 适合长列表，可以减少 DOM 节点数量 90%+

---

### 3. 避免强制同步布局
```javascript
// ❌ 错误：强制同步布局
function badExample() {
  const element = document.getElementById('box')
  element.style.width = '100px'
  const height = element.offsetHeight  // 强制同步布局
  element.style.height = height + 'px'
}

// ✅ 正确：避免读取布局
function goodExample() {
  const element = document.getElementById('box')
  const height = element.offsetHeight  // 先读取
  element.style.width = '100px'
  element.style.height = height + 'px'  // 后写入
}
```

**解释**：
- 强制同步布局会导致浏览器立即计算布局
- 先读取后写入可以避免强制同步布局

## 四、网络优化

### 1. 使用 CDN
```html
<!-- 使用 CDN 加载 Vue -->
<script src="https://cdn.jsdelivr.net/npm/vue@3.3.4/dist/vue.global.min.js"></script>

<!-- 使用 CDN 加载 Tailwind CSS -->
<link rel="stylesheet" href="https://cdn.tailwindcss.com">
```

**解释**：CDN 可以提高资源加载速度，降低服务器负载。

---

### 2. 缓存策略
```javascript
// HTTP 缓存头
Cache-Control: max-age=3600  // 缓存 1 小时
Cache-Control: public, max-age=31536000, immutable  // 永久缓存（用于版本化资源）

// ETag
ETag: "123456789"

// Last-Modified
Last-Modified: Wed, 21 Oct 2026 07:28:00 GMT
```

**解释**：
- Cache-Control 控制缓存时间
- ETag 和 Last-Modified 用于验证资源是否变化
- 版本化资源可以永久缓存（如 `app.v1.2.3.js`）

---

### 3. 压缩传输
```javascript
// 启用 Gzip/Brotli 压缩（服务器配置）
// Nginx
gzip on
gzip_types text/css application/javascript

// Brotli 压缩（更好）
brotli on
brotli_types text/css application/javascript
```

**解释**：Gzip 可以减少 60-80% 的文件大小，Brotli 可以减少 70-85%。

---

### 4. 预连接和预加载
```html
<!-- DNS 预解析 -->
<link rel="dns-prefetch" href="https://api.example.com">

<!-- 预连接（DNS + TCP + TLS 握手） -->
<link rel="preconnect" href="https://api.example.com">

<!-- 预加载资源 -->
<link rel="preload" href="font.woff2" as="font" type="font/woff2" crossorigin>
<link rel="preload" href="image.jpg" as="image">
<link rel="preload" href="styles.css" as="style">

<!-- 预获取（低优先级） -->
<link rel="prefetch" href="next-page.js">
```

**解释**：
- dns-prefetch：提前解析 DNS
- preconnect：提前建立连接
- preload：高优先级加载
- prefetch：低优先级加载（空闲时）

## 五、缓存优化

### 1. Service Worker
```javascript
// 注册 Service Worker
if ('serviceWorker' in navigator) {
  window.addEventListener('load', () => {
    navigator.serviceWorker.register('/sw.js')
  })
}

// sw.js
const CACHE_NAME = 'v1'
const urlsToCache = [
  '/',
  '/styles.css',
  '/script.js',
  '/image.jpg'
]

self.addEventListener('install', event => {
  event.waitUntil(
    caches.open(CACHE_NAME)
      .then(cache => cache.addAll(urlsToCache))
  )
})

self.addEventListener('fetch', event => {
  event.respondWith(
    caches.match(event.request)
      .then(response => response || fetch(event.request))
  )
})
```

**解释**：Service Worker 可以实现离线缓存和资源预加载。

---

### 2. LocalStorage / IndexedDB
```javascript
// LocalStorage（适合小数据）
localStorage.setItem('key', 'value')
const value = localStorage.getItem('key')

// IndexedDB（适合大数据）
const request = indexedDB.open('myDatabase', 1)

request.onsuccess = (event) => {
  const db = event.target.result
  const transaction = db.transaction(['store'], 'readwrite')
  const store = transaction.objectStore('store')
  store.add({ id: 1, name: 'Item' })
}
```

**解释**：
- LocalStorage：适合存储配置、用户偏好等小数据
- IndexedDB：适合存储大量数据（如离线数据）

### 2. React 性能优化

## 九、常见问题

### 1. 什么是 FCP、LCP、FID、CLS？
- **FCP**：首次内容绘制，首次渲染内容的时间
- **LCP**：最大内容绘制，主要内容加载完成的时间
- **FID**：首次输入延迟，首次交互到响应的时间
- **CLS**：累积布局偏移，页面元素意外移动

### 2. 如何减少首屏加载时间？
- 优化图片（WebP、压缩、懒加载）
- 内联关键 CSS
- 代码分割和懒加载
- 使用 CDN
- 预加载关键资源

### 3. 什么是重排和重绘？
- **重排**：重新计算元素位置和大小
- **重绘**：重新绘制元素外观
- 重排比重绘开销大

### 4. 如何优化长列表？
- 虚拟滚动（只渲染可见区域）
- 分页加载
- 延迟加载图片

### 5. 什么是 Service Worker？
- 浏览器后台线程
- 可以实现离线缓存
- 拦截网络请求

### 6. 什么是 Tree Shaking？
- 移除未使用的代码
- 需要 ES Module
- 可以减少打包体积

### 7. 如何减少 JavaScript 体积？
- 按需引入
- 代码分割
- 压缩（Terser）
- Tree Shaking

### 8. 什么是 Key？为什么重要？
- 帮助 Vue/React 识别节点
- 提高 Diff 算法效率
- 避免列表错位

### 9. 什么是 Debounce 和 Throttle？
- **Debounce**：触发后等待一段时间再执行
- **Throttle**：规定时间内只执行一次

### 10. 如何监测性能？
- Lighthouse
- Chrome DevTools
- WebPageTest
- Web Vitals API
