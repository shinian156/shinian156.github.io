## 版本命名规范

采用语义化版本 (Semantic Versioning)：

```
主版本号.次版本号.修订号
   MAJOR    MINOR    PATCH
```

- **MAJOR**: 不兼容的重大更新
- **MINOR**: 向后兼容的新功能
- **PATCH**: 向后兼容的问题修复

## 提交信息规范

使用 Conventional Commits 规范：

```
<type>(<scope>): <subject>

feat: 新功能
fix: 修复bug
docs: 文档更新
style: 代码格式调整
refactor: 重构
test: 测试相关
chore: 构建/工具链相关
```

### 示例
```
feat(post): 添加新的核心知识文章
fix(router): 修复路由跳转问题
docs(readme): 更新项目说明
```
