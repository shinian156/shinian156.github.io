## npm 详解

### 1. 基础命令

```bash
# 初始化项目
npm init                    # 交互式创建 package.json
npm init -y                 # 快速创建 package.json

# 安装包
npm install <package>       # 安装依赖到 dependencies
npm install <package>@<version>  # 安装指定版本
npm install <package> --save-dev  # 安装到 devDependencies
npm install <package> -D         # 同上，简写
npm install                 # 安装 package.json 中的所有依赖
npm install --production    # 只安装 dependencies，不安装 devDependencies

# 更新包
npm update <package>        # 更新指定包
npm update                  # 更新所有包

# 卸载包
npm uninstall <package>     # 卸载包
npm remove <package>         # 同上
npm rm <package>            # 同上
```

### 2. npm 配置

```bash
# 查看配置
npm config list                    # 查看所有配置
npm config get registry            # 查看镜像源
npm config get prefix              # 查看全局安装路径

# 设置配置
npm config set registry https://registry.npmmirror.com  # 设置镜像
npm config set prefix /path/to/global  # 设置全局安装路径
```

### 3. package.json 脚本

```json
{
  "scripts": {
    "dev": "vite",
    "build": "vite build",
    "preview": "vite preview",
    "test": "jest",
    "lint": "eslint src --ext .ts,.vue",
    "lint:fix": "eslint src --fix",
    "format": "prettier --write \"src/**/*.{ts,vue,css}\"",
    "prepare": "husky install"
  }
}
```

```bash
# 运行脚本
npm run dev
npm run build
npm run test -- --coverage  # 传递参数
```

### 4. npm 版本管理

```bash
# 查看包信息
npm view <package>              # 查看包信息
npm view <package> versions      # 查看所有版本
npm view <package> dist-tags    # 查看最新版本标签

# 版本号规则
# major.minor.patch
# 1.2.3
#   |  |  |
#   |  |  └── patch: 补丁版本，修复 bug
#   |  └──── minor: 次版本，新增功能（向后兼容）
#   └─────── major: 主版本，不兼容的改动

# 更新版本
npm version patch   # 1.0.0 -> 1.0.1
npm version minor   # 1.0.0 -> 1.1.0
npm version major   # 1.0.0 -> 2.0.0
```

### 5. npm 依赖类型

```json
{
  "dependencies": {
    "vue": "^3.2.0",      // 生产环境依赖
    "lodash": "~4.17.21", // 兼容版本
    "axios": ">=1.0.0"    // 最低版本
  },
  "devDependencies": {
    "typescript": "^5.0.0",  // 开发环境依赖
    "eslint": "^8.0.0"
  },
  "peerDependencies": {         // 同伴依赖（需用户安装）
    "vue": ">=3.0.0"
  },
  "optionalDependencies": {     // 可选依赖
    "fsevents": "^2.0.0"
  }
}
```

### 6. npm 常用参数

```bash
# 常用参数
npm install --save-exact <package>  # 安装精确版本（不加 ^ ~）
npm install --legacy-peer-deps      # 忽略 peerDependencies 警告
npm install --force                 # 强制重新安装
npm install --ignore-scripts        # 忽略安装脚本
npm install --no-optional           # 不安装 optionalDependencies

# npm install 简写
npm i <package>           # install
npm i -D <package>        # install --save-dev
npm i -g <package>        # 全局安装
npm i -E <package>        # 安装精确版本
```

### 7. npm 全局安装

```bash
# 全局安装
npm install -g <package>

# 查看全局安装路径
npm root -g

# 查看全局安装的包
npm list -g --depth=0

# 更新全局包
npm update -g <package>
npm update -g   # 更新所有全局包

# 卸载全局包
npm uninstall -g <package>
```

## pnpm 详解

### 1. pnpm 特点

- **速度快**: 使用硬链接和符号链接，避免重复下载
- **节省空间**: 同一包的同一版本只存储一份
- **更安全**: 使用沙盒模式，避免恶意代码影响其他项目
- **严格**: 只有 dependencies 会被使用

### 2. 基础命令

```bash
# 安装 pnpm
npm install -g pnpm

# 安装依赖
pnpm install              # 安装所有依赖
pnpm add <package>        # 添加到 dependencies
pnpm add -D <package>     # 添加到 devDependencies
pnpm add -g <package>     # 全局安装

# 更新
pnpm update               # 更新所有
pnpm update <package>      # 更新指定包
pnpm up <package>         # 同 update
pnpm up -r                # 更新所有 workspace

# 卸载
pnpm remove <package>     # 卸载
pnpm rm <package>         # 同上
pnpm uninstall <package>  # 同上
```

### 3. pnpm 高级特性

```bash
# 使用 workspace
pnpm -w <command>         # 在 workspace 根目录执行

# 过滤
pnpm --filter <package> <command>  # 对特定包执行命令

# 清理
pnpm store prune           # 清理未使用的 store
pnpm store status         # 查看 store 状态

# 查看依赖
pnpm list                  # 列出已安装的依赖
pnpm list --depth=0        # 只显示一层依赖
pnpm why <package>         # 查看为什么安装了这个包
```

### 4. pnpm 配置

```bash
# 设置镜像
pnpm config set registry https://registry.npmmirror.com

# pnpm-workspace.yaml
# 在项目根目录创建，定义 workspace
packages:
  - 'packages/*'
  - 'apps/*'
```

### 5. monorepo 配置示例

```yaml
# pnpm-workspace.yaml
packages:
  - 'packages/**'
  - 'apps/**'
```

```json
// packages/shared/package.json
{
  "name": "@myorg/shared",
  "version": "1.0.0",
  "exports": {
    ".": "./index.ts"
  }
}
```

```json
// apps/web/package.json
{
  "name": "@myorg/web",
  "dependencies": {
    "@myorg/shared": "workspace:*"
  }
}
```

## yarn 详解

### 1. yarn 特点

- **速度快**: 并行安装，缓存已下载的包
- **离线安装**: 使用本地缓存，无网也能安装
- **确定性**: yarn.lock 确保安装一致性
- **网络优化**: 扁平化依赖，减少请求数

### 2. 基础命令

```bash
# 安装 yarn
npm install -g yarn

# 初始化
yarn init                  # 交互式创建
yarn init -y              # 快速创建

# 安装
yarn                       # 安装所有依赖
yarn add <package>         # 添加到 dependencies
yarn add -D <package>      # 添加到 devDependencies
yarn add -g <package>      # 全局安装
yarn add --exact <package>  # 安装精确版本

# 更新
yarn upgrade               # 更新所有
yarn upgrade <package>     # 更新指定包
yarn upgrade-interactive    # 交互式更新

# 移除
yarn remove <package>      # 移除包
yarn remove <package> -D   # 移除 devDependencies
```

### 3. yarn 脚本

```bash
# 运行 package.json 中的脚本
yarn <script>
yarn dev
yarn build
yarn test

# 运行其他脚本
yarn run <script>
yarn info <package>         # 查看包信息
yarn outdated              # 检查过时的包
yarn list                  # 列出已安装的包
yarn why <package>         # 查看为什么安装了这个包
```

### 4. yarn 工作区

```json
{
  "workspaces": [
    "packages/*",
    "apps/*"
  ]
}
```

### 5. yarn 特殊命令

```bash
# Yarn 2+ (Berry) 特有命令
yarn dlx <package>         # 临时运行包
yarn plugin import <name>   # 导入插件
yarn workspace <name> <cmd> # 在特定 workspace 执行命令
```

### 6. Yarn PnP (Plug'n'Play)

```bash
# 启用 Yarn PnP
yarn set mode node-modules

# 或使用 zero-install
yarn set mode zero-install
```

## npx 详解

### 1. npx 是什么

npx 是 npm 自带的包执行器，用于运行 npm 包中的命令。

### 2. 基本用法

```bash
# 1. 直接运行包命令（无需全局安装）
npx create-vue@latest       # 创建 Vue 项目
npx create-react-app my-app  # 创建 React 项目
npx tsc                     # 运行 TypeScript 编译器
npx prettier --write .       # 运行 Prettier

# 2. 运行本地 node_modules 中的命令
npx vue-cli-service build    # 运行本地安装的 vue-cli

# 3. 使用特定版本
npx cowsay@1.0.0 "Hello"    # 使用指定版本
npx node@18 index.js         # 使用特定 Node 版本
```

### 3. npx vs npm run

```bash
# npm run 会执行 node_modules/.bin/ 下的脚本
npm run dev

# npx 会在 PATH 和 node_modules/.bin/ 中查找
npx dev  # 如果没有会尝试下载临时执行
```

### 4. npx 参数

```bash
# -p: 指定要安装的包
npx -p <package> <command>

# -c: 使用 package.json 的配置
npx -c '<your-command>'

# --no-install: 如果命令不存在则失败
npx --no-install <command>

# 示例
npx -p typescript tsc --version
```

### 5. 常用 npx 命令

```bash
# 创建项目
npx create-vue my-app
npx create-next-app my-app
npx create-umi my-app
npx degit user/repo my-app

# 代码检查
npx eslint .
npx prettier --write .

# 打包
npx webpack
npx vite build

# 远程执行
npx github:user/repo       # 从 GitHub 下载执行
npx gist:xxxxxx            # 从 Gist 执行
```

### 6. npx vs npm

| 特性 | npx | npm |
|------|-----|-----|
| 用途 | 执行命令 | 管理包 |
| 全局安装 | 不需要 | 需要 |
| 临时执行 | 支持 | 不支持 |
| 包缓存 | 临时缓存 | 持久缓存 |
| 版本指定 | 支持 | 不直接支持 |

## 工具对比

### 功能对比

| 特性 | npm | pnpm | yarn | yarn (Berry) |
|------|-----|------|------|-------------|
| 安装速度 | 中等 | 快 | 快 | 快 |
| 磁盘空间 | 较大 | 小 | 中等 | 小 |
| 离线安装 | 不支持 | 支持 | 支持 | 支持 |
| lock 文件 | package-lock.json | pnpm-lock.yaml | yarn.lock | .pnp.cjs |
| Monorepo | 需要 workspace | 原生支持 | 需要 workspace | 原生支持 |
| 幽灵依赖 | 存在 | 不存在 | 存在 | 不存在 |

### 命令对比

| 操作 | npm | pnpm | yarn |
|------|-----|------|------|
| 安装 | `npm i` | `pnpm i` | `yarn` |
| 添加 | `npm i pkg` | `pnpm add pkg` | `yarn add pkg` |
| 移除 | `npm rm pkg` | `pnpm rm pkg` | `yarn remove pkg` |
| 更新 | `npm up pkg` | `pnpm up pkg` | `yarn up pkg` |
| 全局 | `npm i -g pkg` | `pnpm add -g pkg` | `yarn global add pkg` |
| 执行 | `npx cmd` | `pnpm exec cmd` | `yarn dlx cmd` |

### 速度对比

```
pnpm > yarn > npm

pnpm: 使用硬链接，速度最快
yarn: 并行安装，有缓存
npm: 串行安装，相对较慢
```

## 最佳实践

### 1. 选择合适的包管理器

```bash
# 新项目推荐使用 pnpm
npm install -g pnpm

# 或 yarn
npm install -g yarn
```

### 2. 项目初始化

```bash
# 初始化项目
pnpm init

# 添加依赖
pnpm add vue vue-router pinia
pnpm add -D typescript vite @vitejs/plugin-vue

# 锁定文件
pnpm import  # 从 yarn.lock 或 package-lock.json 转换
```

### 3. CI/CD 配置

```bash
# .github/workflows/ci.yml
- name: Install dependencies
  run: pnpm install --frozen-lockfile

# 或
- name: Install dependencies
  run: npm ci  # npm 专用，干净安装
```

### 4. 切换镜像源

```bash
# npm
npm config set registry https://registry.npmmirror.com

# pnpm
pnpm config set registry https://registry.npmmirror.com

# yarn
yarn config set registry https://registry.npmmirror.com
```

### 5. 常见问题解决

```bash
# 清理缓存
npm cache clean --force
pnpm store prune
yarn cache clean

# 删除 node_modules 重新安装
rm -rf node_modules
npm install

# 修复损坏的安装
npm rebuild
pnpm rebuild
yarn rebuild
```

### 6. 开发团队统一工具

```bash
# 在 .nvmrc 或 .node-version 文件中指定 Node 版本
16.20.0

# packageManager 字段 (Node.js 16.10+)
{
  "packageManager": "pnpm@8.0.0"
}
```

## 总结

| 工具 | 适用场景 |
|------|---------|
| **npm** | 通用，推荐作为默认选择 |

| **yarn** | 需要 yarn.lock 确保一致性 |
| **npx** | 临时执行命令、创建项目 |
