## 行为准则

参与本项目即表示您同意遵守以下行为准则：

1. **尊重他人** - 保持友善和专业的沟通态度
2. **包容多样性** - 欢迎不同背景和经验的贡献者
3. **建设性反馈** - 提供具体、有帮助的反馈
4. **专注目标** - 讨论聚焦于项目本身

## 开始贡献

### 环境准备

```bash
# 1. Fork 本仓库
# 访问 https://github.com/your-username/frontend-blog 点击 Fork 按钮

# 2. 克隆你的 Fork 到本地
git clone https://github.com/your-username/frontend-blog.git

# 3. 进入项目目录
cd frontend-blog

# 4. 添加上游仓库
git remote add upstream https://github.com/original-owner/frontend-blog.git

# 5. 安装依赖
npm install

# 6. 启动开发服务器
npm run dev
```

### 创建分支

```bash
# 确保从最新的 main 分支创建
git checkout main
git pull upstream main

# 创建新分支
git checkout -b feature/your-feature-name
# 或
git checkout -b fix/your-bug-fix
```

## 提交流程

### 1. 开发

```bash
# 在新分支上完成你的开发
git checkout -b feature/add-new-article
```

### 2. 提交

```bash
# 添加文件到暂存区
git add .

# 提交（遵循提交规范）
git commit -m 'feat(post): 添加 Vue3 组合式 API 核心知识'

# 推送到你的 Fork
git push origin feature/add-new-article
```

### 3. 创建 Pull Request

1. 访问你的 Fork 仓库
2. 点击 "Compare & pull request"
3. 填写 PR 描述
4. 提交 PR

### 4. PR 审查

- 维护者会在 1-3 天内审查您的 PR
- 可能需要根据反馈进行修改
- 审核通过后，您的代码将被合并

## 代码规范

### JavaScript / TypeScript

```typescript
// 使用 TypeScript，提供完整的类型定义
interface Article {
  id: string
  title: string
  category: string
  content: string
  tags?: string[]
}

// 组件使用 Composition API
<script setup lang="ts">
import { ref, computed } from 'vue'

const count = ref(0)
const doubled = computed(() => count.value * 2)
</script>
```

### Vue 组件规范

```vue
<template>
  <!-- 使用语义化标签 -->
  <article class="article">
    <header>
      <h1>{{ title }}</h1>
    </header>
    <main v-html="content"></main>
  </article>
</template>

<script setup lang="ts">
// 类型导入
import { defineProps } from 'vue'

// 定义 props
interface Props {
  title: string
  content: string
}

const props = defineProps<Props>()
</script>

<style scoped>
/* 使用 scoped 样式 */
.article {
  max-width: 900px;
  margin: 0 auto;
}
</style>
```

### CSS 规范

```css
/* 使用 BEM 命名或原子化 CSS */
.article-card {
  padding: 1rem;
  border-radius: 8px;
}

.article-card__title {
  font-size: 1.25rem;
}

.article-card--highlight {
  background: #f0f0f0;
}

/* 使用 CSS 变量 */
:root {
  --primary-color: #42b883;
  --spacing-unit: 1rem;
}
```

### Git 提交规范

```
<type>(<scope>): <subject>

feat(post): 添加新的核心知识文章
fix(router): 修复路由跳转问题
docs(readme): 更新项目说明
style(home): 优化首页布局
refactor(article): 重构文章组件
test(store): 添加单元测试
chore(deps): 更新依赖版本
```

#### Type 类型

| 类型 | 说明 |
|------|------|
| feat | 新功能 |
| fix | 修复 bug |
| docs | 文档更新 |
| style | 代码格式调整（不影响功能） |
| refactor | 重构（不是新功能也不是修复 bug） |
| perf | 性能优化 |
| test | 添加测试 |
| chore | 构建/工具链相关 |

## 文档贡献

### 文档格式

文档使用 Markdown 编写，请遵循以下格式：

```markdown
# 标题

## 简介
简要说明...

## 目录
- [章节1](#章节1)
- [章节2](#章节2)

---

## 章节1

内容...

### 子章节
更多内容...
```

### 文档命名

- 文件名使用小写和连字符
- 示例：`contributing-guide.md`、`api-documentation.md`

## 核心知识文章格式

### 文件命名

核心知识文章按以下格式命名：

```
posts/
├── 01-Vue.md
├── 02-uni-app.md
└── ...
```

### 文章结构

```markdown
# [标题] 核心知识

## 目录
1. [问题1](#问题1)
2. [问题2](#问题2)

---

## 问题1

### 问题描述
这是问题的描述...

### 答案
这是详细的答案...

### 代码示例
```javascript
// 示例代码
const example = 'hello'
```

### 知识点
- 知识点1
- 知识点2

---

## 问题2

...（同上）
```

### 文章要求

1. **标题清晰** - 使用 `#` 标记主标题
2. **层次分明** - 使用 `##` 和 `###` 标记章节
3. **代码完整** - 所有代码示例必须可运行
4. **解释详细** - 不仅给出答案，还要解释原因
5. **标注重点** - 使用加粗、列表等标记重点

## 问题反馈

### Bug 报告

请在提交 Bug 报告时包含：

1. **问题简述** - 一句话描述问题
2. **复现步骤** - 清晰的步骤来复现问题
3. **预期行为** - 期望的正确行为
4. **实际行为** - 实际发生的错误
5. **环境信息**：
   - 操作系统
   - 浏览器版本
   - Node.js 版本
   - npm 版本

### 功能建议

请在提交功能建议时包含：

1. **功能描述** - 详细描述建议的功能
2. **使用场景** - 这个功能解决什么问题
3. **可能的实现方案** - 您对如何实现的初步想法

### Issue 模板

- [ ] 其他
