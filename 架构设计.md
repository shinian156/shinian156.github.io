### 目录职责

#### `src/` - 源代码目录
- **components/** - 可复用的UI组件
- **views/** - 页面级组件（路由对应）
- **router/** - 路由配置和守卫
- **App.vue** - 根组件，布局容器
- **main.ts** - 应用入口，挂载点

#### `posts/` - 内容目录
- 存放所有核心知识Markdown文件
- 20份按主题分类的文章
- 支持直接Markdown渲染

#### `public/` - 公共资源
- 不经过构建处理的静态资源
- 直接映射到服务器根路径

#### `assets/` - 资源目录
- 存放图片、字体等经过构建处理的资源
- Vite 会处理这些文件

## 核心模块设计

### 1. 根组件 (App.vue)

**职责**：
- 作为应用的顶层容器
- 整合所有公共组件（Header、Sidebar、Footer）
- 提供页面布局结构

**设计模式**：
```vue
<template>
  <div id="app">
    <Header />
    <div class="main-container">
      <Sidebar />
      <main class="content">
        <router-view />
      </main>
    </div>
    <Footer />
  </div>
</template>
```

### 2. 路由配置 (router/index.ts)

**职责**：
- 定义应用的所有路由
- 配置路由守卫（如需要）
- 管理路由元信息

**路由表设计**：
```typescript
const routes = [
  {
    path: '/',
    name: 'home',
    component: () => import('@/views/Home.vue'),
    meta: { title: '首页' }
  },
  {
    path: '/article/:id',
    name: 'article',
    component: () => import('@/views/Article.vue'),
    meta: { title: '文章详情' }
  }
]
```

### 3. 首页视图 (Home.vue)

**职责**：
- 展示所有技术分类
- 展示文章列表
- 响应用户交互

**组件结构**：
```
Home.vue
├── Hero Section (统计信息)
├── Category Grid (分类卡片网格)
│   ├── Category Card 1
│   ├── Category Card 2
│   └── ...
└── Quick Links (快速链接)
```

### 4. 文章视图 (Article.vue)

**职责**：
- 加载和渲染Markdown文章
- 展示文章元信息
- 提供导航功能

**数据流**：
```
URL参数 → 加载文章 → Markdown解析 → HTML渲染 → 代码高亮
```

## 状态管理

### Pinia Store 设计

虽然当前项目主要使用组件本地状态，但为未来扩展预留了 Pinia 状态管理：

```typescript
// src/stores/article.ts (预留)
import { defineStore } from 'pinia'
import { ref, computed } from 'vue'

export const useArticleStore = defineStore('article', () => {
  // 状态
  const articles = ref<Article[]>([])
  const currentArticle = ref<Article | null>(null)

  // Getter
  const getAllArticles = computed(() => articles.value)

  // Action
  const setCurrentArticle = (article: Article) => {
    currentArticle.value = article
  }

  return {
    articles,
    currentArticle,
    getAllArticles,
    setCurrentArticle
  }
})
```

### 状态管理扩展

如需添加全局状态，可在 `src/stores/` 目录下创建新的 store 文件。

## 扩展性设计

### 1. 添加新页面

1. 在 `src/views/` 创建新组件
2. 在 `src/router/index.ts` 添加路由
3. 可选：在 Pinia store 中添加状态

### 2. 添加新组件

1. 在 `src/components/` 创建组件
2. 在需要的视图中导入使用
3. 如需全局注册，在 `main.ts` 中引入

### 3. 添加新内容

1. 在 `posts/` 创建 Markdown 文件
2. 在 Home.vue 的分类配置中添加链接
3. 确保文件名唯一

### 4. 扩展功能

#### 搜索功能
```typescript
// src/utils/search.ts
export const searchArticles = (query: string, articles: Article[]) => {
  return articles.filter(article =>
    article.title.toLowerCase().includes(query.toLowerCase()) ||
    article.content.toLowerCase().includes(query.toLowerCase())
  )
}
```

#### 收藏功能
```typescript
// src/stores/favorite.ts
export const useFavoriteStore = defineStore('favorite', {
  state: () => ({
    favorites: [] as string[]
  }),
  actions: {
    toggleFavorite(articleId: string) {
      const index = this.favorites.indexOf(articleId)
      if (index > -1) {
        this.favorites.splice(index, 1)
      } else {
        this.favorites.push(articleId)
      }
    }
  }
})
```

## 安全性考虑

### XSS 防护

文章内容通过 `marked` 解析后使用 `v-html` 渲染，需要确保：
1. 用户输入的文章内容经过净化
2. 使用 DOMPurify 等库清理 HTML

### CSP 配置

```html
<meta http-equiv="Content-Security-Policy" content="default-src 'self'; script-src 'self' 'unsafe-inline'; style-src 'self' 'unsafe-inline';">
```
